# Contributing to Orca 🐋

First off, thank you for considering contributing to Orca! It’s people like you that make Orca a better tool for everyone.

## How to Contribute

There are many ways to contribute, and they are all appreciated:

- **Bug Reports**: If you find something that isn't working right, please open an issue. Try to be as descriptive as possible.
- **Feature Requests**: Have an idea for a new feature? We'd love to hear it!
- **Code Changes**: If you're looking to dive into the code, feel free to open a Pull Request.
- **Documentation**: Fixing typos or improving the clarity of the documentation is always helpful.

## Our Philosophy

Orca aims to be a polite and helpful ecosystem. When contributing:
1. **Be Respectful**: We value diversity of opinion and background.
2. **Focus on Quality**: We prefer clean, readable code over clever hacks.
3. **Rust Best Practices**: Try to follow standard Rust idioms and conventions.

## Development Workflow

1. Fork the repo and create your branch from `main`.
2. Ensure `cargo check` passes with no warnings.
3. If you've changed the UI, please include a screenshot in your PR!

## Questions?

If you have questions about the codebase or the project goals, don't hesitate to open an issue or reach out. No question is too small.

Happy coding! 🐋✨
